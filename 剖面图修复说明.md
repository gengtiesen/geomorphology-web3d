# 剖面图读取问题修复说明

## 问题描述

用户反馈剖面图没有能够成功读取。

## 问题分析

经过检查发现两个问题：

1. **文件名不匹配**：实际文件名是 `观测点1  剖面图.jpg`（两个空格），但代码中使用的是 `观测点1 剖面图.jpg`（一个空格）

2. **类型不匹配**：用户修改了 `fileUtils.ts` 将 `profileImage` 改为数组类型 `string[]`，但类型定义中仍是 `string`

## 修复方案

### 1. 修复类型定义

**文件**: `src/types/index.ts`

```typescript
// 修改前
profileImage?: string;    // 剖面图路径

// 修改后  
profileImage?: string[];  // 剖面图路径数组
```

### 2. 修复文件名匹配

**文件**: `src/utils/fileUtils.ts`

```typescript
// 修改前
const profileResponse = await fetch(`${basePath}${pointName} 剖面图.jpg`);

// 修改后（注意两个空格）
const profileResponse = await fetch(`${basePath}${pointName}  剖面图.jpg`);
```

### 3. 更新UI组件

**文件**: `src/components/Sidebar.tsx`

```typescript
// 修改前
{selectedObservationPoint.profileImage && (
  <Panel header="剖面图" key="profile">
    <Image src={selectedObservationPoint.profileImage} />
  </Panel>
)}

// 修改后（支持数组显示）
{selectedObservationPoint.profileImage && selectedObservationPoint.profileImage.length > 0 && (
  <Panel header="剖面图" key="profile">
    <Image.PreviewGroup>
      {selectedObservationPoint.profileImage.map((profileImg, index) => (
        <Image key={index} src={profileImg} alt={`剖面图${index + 1}`} />
      ))}
    </Image.PreviewGroup>
  </Panel>
)}
```

## 使用说明

现在用户可以：

1. 启动应用：`npm run dev`
2. 点击"加载示例数据"按钮
3. 点击地图上的观测点或侧边栏中的观测点卡片
4. 在弹出的详情对话框中查看：
   - ✅ 知识点内容
   - ✅ 剖面图（支持多张图片预览）
   - ✅ 实地图片

## 技术改进

1. **支持数组类型**：现在 `profileImage` 支持多张剖面图
2. **图片预览组**：使用 `Image.PreviewGroup` 提供更好的用户体验
3. **错误处理**：保留了完整的错误处理和警告机制
4. **调试信息**：添加了console.log便于调试

## 注意事项

1. 文件命名必须精确匹配，包括空格数量
2. 支持的图片格式：JPG、PNG
3. 文件必须放在 `public/观测点文档/` 目录下
4. 首次加载时会有异步请求，可能有短暂延迟
