# 故障排除指南

## 常见问题及解决方案

### 1. Cesium 地图无法加载

**症状**：页面空白或显示错误信息

**可能原因**：
- 网络连接问题
- Cesium Ion 访问令牌无效
- 浏览器不支持 WebGL

**解决方案**：
1. 检查网络连接
2. 打开浏览器开发者工具，查看控制台错误信息
3. 验证 WebGL 支持：访问 [webglreport.com](https://webglreport.com/)
4. 更新浏览器到最新版本
5. 启用硬件加速

### 2. 轨迹文件导入失败

**症状**：导入 GPX 或 JSON 文件时显示错误

**可能原因**：
- 文件格式不正确
- 文件损坏
- 文件过大

**解决方案**：
1. 确认文件格式为标准 GPX 1.1 或正确的 JSON 格式
2. 检查文件是否完整，没有截断
3. 尝试使用较小的文件进行测试
4. 查看浏览器控制台的详细错误信息

### 3. File System Access API 不可用

**症状**：无法选择工作目录或保存文件

**可能原因**：
- 浏览器不支持 File System Access API
- 在不安全的上下文中运行（非 HTTPS）

**解决方案**：
1. 使用支持的浏览器（Chrome 86+, Edge 86+）
2. 确保在 HTTPS 环境下运行
3. 在本地开发时使用 localhost
4. 作为备选，使用传统的文件上传/下载功能

### 4. 性能问题

**症状**：地图渲染缓慢或卡顿

**可能原因**：
- 硬件配置不足
- 大量数据加载
- 浏览器性能限制

**解决方案**：
1. 关闭其他占用资源的程序
2. 减少同时显示的轨迹数量
3. 降低地图详细程度
4. 使用性能更好的设备

### 5. 轨迹显示异常

**症状**：轨迹不显示或显示位置错误

**可能原因**：
- 坐标系统错误
- 数据格式问题
- 高程数据缺失

**解决方案**：
1. 确认坐标使用 WGS84 地理坐标系
2. 检查经纬度数值是否在有效范围内
3. 验证 GPX 文件的结构是否正确
4. 尝试重新导入数据

### 6. 观察点标记不显示

**症状**：导入观察点后在地图上看不到标记

**可能原因**：
- 坐标格式错误
- 标记图标缺失
- 视角问题

**解决方案**：
1. 检查坐标文件格式：`经度,纬度,高程,名称`
2. 确认坐标数值正确
3. 使用"飞行到"功能定位到观察点
4. 检查标记是否被地形遮挡

## 开发者调试

### 启用详细日志

在浏览器控制台中运行：
```javascript
// 启用 Cesium 调试信息
window.CESIUM_BASE_URL = '/cesium/';
```

### 常用调试命令

```javascript
// 获取当前相机位置
viewer.camera.positionCartographic

// 获取当前视角参数
viewer.camera.heading
viewer.camera.pitch
viewer.camera.roll

// 清除所有实体
viewer.entities.removeAll()

// 检查 WebGL 上下文
viewer.scene.context._gl
```

### 性能监控

启用 Cesium 性能统计：
```javascript
viewer.scene.debugShowFramesPerSecond = true;
```

## 获取帮助

如果以上解决方案都无法解决问题：

1. **检查浏览器控制台**：记录所有错误信息
2. **记录操作步骤**：详细描述如何重现问题
3. **系统信息**：包括操作系统、浏览器版本、硬件配置
4. **创建 Issue**：在项目仓库中创建问题报告

## 系统要求

### 最低要求
- 现代浏览器（Chrome 80+, Firefox 75+, Safari 13+, Edge 80+）
- WebGL 2.0 支持
- 4GB RAM
- 集成显卡

### 推荐配置
- Chrome 100+ 或 Edge 100+
- 8GB+ RAM
- 独立显卡
- SSD 存储

## 已知限制

1. **文件大小**：单个轨迹文件建议不超过 10MB
2. **浏览器兼容性**：File System Access API 仅在部分浏览器中可用
3. **移动端**：移动设备上的性能可能受限
4. **网络依赖**：地形和影像数据需要网络连接
