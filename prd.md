# 地貌实习可视化Web3D平台 - 产品需求文档

## 项目概述
开发一个基于Web的三维地貌实习可视化平台，为地质、地理、环境科学等专业的学生和研究人员提供沉浸式的野外实习体验和数据记录工具。

## 核心技术栈
地图开发平台需要用Cesium作为核心3D地球引擎

## 功能需求

### 1. 三维地图基础功能
- **三维底图显示**
  - 支持多种底图源（卫星影像、地形图、街道图等）
  - 地形渲染与高程数据可视化
  - 多级别缩放（从全球视角到局部详细视图）
  - 支持倾斜摄影和真实感渲染

### 2. 轨迹数据管理
- **实时轨迹记录**
  - GPS定位数据采集
  - 实时轨迹绘制和显示
  - 轨迹数据导入/导出（GPX、KML格式）
  - 历史轨迹回放功能
  
- **轨迹分析工具**
  - 距离、速度、海拔变化统计
  - 轨迹优化和平滑处理
  - 多条轨迹对比分析

### 3. 高精度DEM数据处理
- **DEM数据叠加**
  - 支持多种DEM数据格式（GeoTIFF、HGT等）
  - 高精度地形渲染
  - 等高线生成和显示
  - 坡度、坡向分析
  
- **地形分析工具**
  - 剖面线绘制和高程剖面图
  - 可视域分析
  - 流域分析工具

### 4. 实习笔记系统
- **多媒体笔记**
  - 文字笔记编辑（支持Markdown格式）
  - 图片上传和标注
  - 语音录制和播放
  - 视频片段记录
  
- **地理标注功能**
  - 在3D地图上精确定位笔记
  - 自定义标记图标和样式
  - 笔记分类和标签管理
  - 笔记搜索和筛选

### 5. 数据管理功能（本地存储）
- **本地数据存储**
  - 使用IndexedDB存储大容量数据
  - LocalStorage存储用户配置
  - 数据导入/导出功能
  - 数据备份和恢复
  
- **离线模式支持**
  - 缓存地图瓦片
  - 离线数据访问
  - 数据同步机制（为后续云端迁移预留）

## 技术架构（纯前端方案）

### 前端技术栈
- **核心框架**: React 18 + TypeScript + Vite
- **3D引擎**: Cesium.js
- **UI组件库**: Ant Design 或 Material-UI
- **状态管理**: Zustand（轻量级状态管理）
- **地图工具**: Turf.js（地理空间分析）
- **文件处理**: FileSaver.js, Papa Parse
- **本地文件访问**: 
  - File System Access API（现代浏览器）
  - 传统文件上传/下载（兼容性备选）

### 数据存储方案（本地文件系统）
- **轨迹数据**: 
  - 存储格式：GPX/GeoJSON文件
  - 存储位置：用户指定的本地文件夹
  - 文件命名：`track_YYYY-MM-DD_HH-mm-ss.gpx`
  
- **笔记数据**: 
  - 文本笔记：JSON文件存储 `notes_YYYY-MM-DD.json`
  - 图片文件：原始格式存储在 `images/` 子文件夹
  - 语音文件：存储在 `audio/` 子文件夹
  
- **DEM数据**: 
  - 直接读取用户上传的GeoTIFF文件
  - 临时处理数据存储在浏览器缓存
  
- **用户配置**: 
  - LocalStorage存储界面设置和偏好
  - 工作目录路径记录

### 文件组织结构
```
工作目录/
├── tracks/           # 轨迹文件
│   ├── track_2024-01-15_09-30-00.gpx
│   └── track_2024-01-15_14-20-00.json
├── notes/            # 笔记文件
│   ├── notes_2024-01-15.json
│   ├── images/       # 图片文件夹
│   │   ├── IMG_001.jpg
│   │   └── IMG_002.png
│   └── audio/        # 音频文件夹
│       └── REC_001.wav
├── dem/              # DEM数据文件
│   └── area_dem.tiff
└── config/           # 配置文件
    └── settings.json
```

### 数据格式支持
- **轨迹数据**: GPX, GeoJSON, KML
- **影像数据**: GeoTIFF, PNG/JPEG
- **矢量数据**: GeoJSON, KML
- **DEM数据**: GeoTIFF格式

## 用户界面设计

### 主界面布局
- **左侧面板**: 图层控制、工具箱、本地文件列表
- **中央区域**: 3D地球视图
- **右侧面板**: 属性面板、详细信息显示
- **底部状态栏**: 坐标显示、比例尺、当前工作目录

### 文件管理界面
- **工作目录选择**: 用户可选择本地文件夹作为工作目录
- **文件浏览器**: 显示当前目录下的所有数据文件
- **文件操作**: 导入、导出、删除、重命名功能
- **自动扫描**: 启动时自动扫描工作目录中的数据文件

### 移动端适配
- 响应式设计，支持平板和手机
- 触摸手势操作（缩放、旋转、平移）
- 移动端使用传统文件上传方式
- GPS定位和传感器数据集成

## 开发计划

### 第一阶段
1. **基础3D地图搭建**
   - Cesium环境配置和基础场景
   - 底图服务集成（OpenStreetMap、Bing等）
   - 相机控制和导航
   - 基础UI框架搭建

2. **文件系统集成**
   - File System Access API集成
   - 工作目录选择和管理
   - 文件读取和写入功能
   - 文件格式解析（GPX/GeoJSON）

3. **轨迹功能实现**
   - GPS轨迹记录和显示
   - 轨迹文件保存和加载
   - 基础轨迹分析（距离、时间）

### 第二阶段
1. **笔记系统开发**
   - 文字笔记编辑器
   - 图片文件保存和加载
   - 地理位置绑定
   - 笔记文件管理

2. **DEM数据处理**
   - GeoTIFF文件读取
   - DEN地形数据可视化

3. **文件管理功能**
   - 文件浏览和组织
   - 批量导入导出
   - 数据备份功能

### 兼容性要求
- 支持Chrome, Firefox, Safari, Edge最新版本
- 移动端支持iOS 12+, Android 8+
- WebGL 2.0支持要求
- File System Access API支持（现代浏览器）
- 传统文件API兼容（旧版浏览器）


### 数据保存流程
1. 用户记录轨迹或添加笔记
2. 实时保存到本地文件
3. 自动生成文件名和组织结构
4. 更新文件列表显示
