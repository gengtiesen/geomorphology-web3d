# 地貌实习可视化Web3D平台 - 使用指南

## 快速开始

### 1. 启动应用

```bash
npm install
npm run dev
```

应用将在 http://localhost:5173 启动

### 2. 界面概览

- **左侧边栏**：数据管理和轨迹列表
- **主视图**：3D地球和地图显示
- **右上角**：图层选择器

## 主要功能

### 📍 加载示例数据

1. 点击左侧边栏的"加载示例数据"按钮
2. 系统会自动加载大同地区的观察点和实习路径
3. 地图会显示红色观察点标记和蓝色轨迹线

### 📂 导入轨迹文件

**支持格式**：GPX、JSON

1. 点击"导入轨迹"按钮
2. 选择 GPX 或 JSON 格式的轨迹文件
3. 系统自动解析并在地图上显示
4. 轨迹会自动贴合地形表面

**GPX 文件要求**：
- 标准 GPX 1.1 格式
- 包含轨迹点的经纬度信息
- 可选：高程和时间信息

**JSON 文件格式**：
```json
{
  "id": "track_id",
  "name": "轨迹名称",
  "points": [
    {
      "lng": 113.677983,
      "lat": 40.105708,
      "elevation": 1227,
      "time": "2024-01-15T09:00:00.000Z"
    }
  ],
  "color": "#ff0000",
  "visible": true
}
```

### 🎨 轨迹管理

**颜色调整**：
1. 在轨迹管理面板中找到目标轨迹
2. 点击轨迹卡片左侧的颜色按钮
3. 选择预设颜色或自定义颜色
4. 颜色会实时更新到地图上

**显示控制**：
- 点击眼睛图标切换轨迹显示/隐藏
- 点击"定位"按钮飞行到轨迹位置
- 点击下载图标导出轨迹为 GPX 格式

**删除轨迹**：
- 点击删除图标（垃圾桶）
- 确认删除操作

### 📍 导入观察点

**支持格式**：TXT、CSV

1. 点击"导入观察点"按钮
2. 选择包含坐标的文本文件
3. 系统解析并在地图上显示红色标记

**文件格式要求**：
```
经度,纬度,高程,名称
113.677983,40.105708,1227,观察点1
113.677172,40.106972,1228,观察点2
```

### 💾 文件系统集成（现代浏览器）

**选择工作目录**：
1. 点击"选择工作目录"按钮
2. 选择本地文件夹作为工作目录
3. 系统会记住该目录用于数据保存

**支持的浏览器**：
- Chrome 86+
- Edge 86+
- 其他浏览器将使用传统文件上传/下载

### 🗺️ 地图操作

**基本操作**：
- **鼠标左键拖拽**：平移地图
- **鼠标滚轮**：缩放
- **鼠标右键拖拽**：旋转视角
- **Shift + 左键拖拽**：倾斜视角

**图层选择**：
1. 点击右上角的图层选择器
2. 选择不同的底图：
   - Bing Maps Aerial（卫星影像）
   - Bing Maps Road（道路地图）
   - OpenStreetMap（开源地图）
   - Natural Earth II（自然地球影像）

**地形功能**：
- 高精度 30 米分辨率地形
- 真实地形阴影和光照
- 轨迹自动贴合地形表面

## 数据格式说明

### GPX 格式示例

```xml
<?xml version="1.0" encoding="UTF-8"?>
<gpx version="1.1" creator="地貌实习可视化Web3D平台">
  <trk>
    <name>实习轨迹</name>
    <trkseg>
      <trkpt lat="40.105708" lon="113.677983">
        <ele>1227</ele>
        <time>2024-01-15T09:00:00.000Z</time>
      </trkpt>
    </trkseg>
  </trk>
</gpx>
```

### 观察点坐标文件示例

```
113.677983,40.105708,1227,观察点1
113.677172,40.106972,1228,观察点2
113.676361,40.108236,1230,观察点3
```

## 实用技巧

### 🎯 精确定位

1. **使用示例数据学习**：先加载示例数据熟悉界面
2. **轨迹定位**：使用"定位"功能快速找到轨迹位置
3. **多角度观察**：使用鼠标右键旋转查看不同角度

### 📊 数据分析

1. **距离统计**：轨迹卡片显示总距离和点数
2. **高程信息**：观察轨迹的高程变化
3. **时间序列**：GPX 文件包含时间信息时可查看时间序列

### 🎨 视觉效果

1. **颜色编码**：为不同类型的轨迹使用不同颜色
2. **图层组合**：尝试不同底图查看最佳效果
3. **光照效果**：注意地形的光照和阴影效果

### 💡 性能优化

1. **分批加载**：避免同时加载过多轨迹
2. **隐藏不需要的轨迹**：使用显示/隐藏功能
3. **适当缩放**：避免过度放大影响性能

## 常见问题

### Q: 轨迹导入后看不到？
A: 检查轨迹是否被隐藏，或使用"定位"功能飞行到轨迹位置。

### Q: 文件格式不支持？
A: 确保使用标准 GPX 1.1 格式或正确的 JSON 结构。

### Q: 性能卡顿？
A: 减少同时显示的轨迹数量，关闭不必要的图层。

### Q: 无法选择工作目录？
A: 使用支持 File System Access API 的浏览器（Chrome 86+, Edge 86+）。
